Stateful Components in Vanilla JS

Today I‚Äôd like to show you how you can create stateful componenents with vanilla JavaScript üç¶, and demonstrate that:

You don‚Äôt need to download any library to apply the amazing concepts used by React.js in your JavaScript programming.

Yes, it‚Äôs now entirely possible to use the powerful features of JavaScript out-of-the-box to do some pretty amazing things. Let‚Äôs dive right in!

Start

Let‚Äôs start with a simple object, which will represent the initial ‚Äústate‚Äù of our data:

let state = {
  count: 0,
};

From here, I will create several methods, which I will divide into 2 categories:

  1. Component functions
  2. Render functions

We will use ES6 template strings (See: Template Literals ‚ÜóÔ∏è) to construct our components, which can be fed data and react dynamically to user actions.

Render functions will be called to display the component when the user triggers changes.

First let‚Äôs create our counter component, which is just an arrow function that returns a template string.

const counter = (count) => {
  return `<div class="counter">${count}</div>;
}

Then let‚Äôs create a render function, which we can call to update the DOM when a user triggers a change to the count.

renderCount = () => {
  document.getElementById('app').innerHTML = counter(this.state.count);
};

Now, all we need is a method to update the count. Since we don‚Äôt have a setState({}) method built-in to JavaScript just yet, we will do a React no-no, and set the state directly üôÄ:

incrementCountUp = () => {
  let newCount = this.state.count + 1;
  this.state.count = newCount;
};

Then we can attach an event listener to a button click.

When the user clicks the button, the count will increment up by updating the value of state.count. After that, we can call the render method so we will instantly see the change rendered in the DOM:

const incCountBtn = document.querySelector("[data-action='count-up']");

incCountBtn.click(() => {
  incrementCountUp();
  renderCount();
});

And with that, we have a stateful component created with plain-old vanilla JS!

Bonus material

Just like in React.js, we can utilize JS logic within our components to modify the frontend user experience.

For example, we can change the class name in use by checking the value of count:

const counter = (count) => {
  let className = "text-error";

  if (count > 10) {
    classname = "text-success";
  }

  return `<div class="${className}">${count}</div>;
}

And in our CSS:

.text-error {
  color: indianred;
}

.text-success {
  color: green;
}

Now, when we increment our count past 10, the color of the text will automatically change, without having to do any extra DOM manipulation wacky jQuery stuff.

Conclusion

As you can see, it is very handy to program in JavaScript this way, using the powerful features built-in to the syntax. Even though we can‚Äôt benefit from React‚Äôs virtual DOM or setState({}) method, this is a powerful way to think and program ‚Äî and there is no external dependency to worry about!

What are the benefits?

  1. Write organized, reactive code without the increased bundle size
  2. Simplify DOM manipulations and state logic
  3. Improve the versatility and strength of your standard JavaScript coding!
